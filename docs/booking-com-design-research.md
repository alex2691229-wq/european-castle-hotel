# Booking.com 後台設計研究

## 核心設計元素

### 1. 日曆視圖 (Calendar View)
- **列表視圖 (List View)**: 顯示所有房型，每個房型有「Rooms to sell」和「Rate plan」行
- **月視圖 (Monthly View)**: 顯示單一房型的月曆，可以拖曳選擇日期範圍

### 2. 關鍵功能行
- **Rooms to sell (可銷售房間數)**: 設置每個日期可銷售的房間數量
- **Rate plan (價格方案)**: 設置每個日期的價格

### 3. 批量編輯功能 (Bulk Edit)
- 可以一次更改多個日期
- 可以一次更改多個房型
- 彈出面板讓用戶選擇日期範圍、房間數量、價格

### 4. 日期範圍選擇
- 點擊並拖曳選擇多個日期
- 進階日期選擇：可以指定只更改特定的星期幾

### 5. 狀態顏色指示
- **綠色**: 房間開放且可預訂
- **紅色**: 房間不可預訂

### 6. 不可預訂原因
- Closed (已關閉)
- No inventory (無庫存)
- Price missing (缺少價格)
- Rate closed (價格方案已關閉)
- Multiple blockers (多個問題)
- Sold out (售罄)

## 設計原則

1. **統一界面**: 價格和可用性在同一個日曆界面管理
2. **視覺化狀態**: 使用顏色區分不同狀態
3. **批量操作**: 支持多日期、多房型的批量編輯
4. **拖曳選擇**: 直觀的日期範圍選擇方式
5. **即時反饋**: 清楚顯示為什麼房間不可預訂

## 應用到我們的系統

### 建議的統一房間管理界面結構

1. **頂部**: 房型選擇下拉菜單 + 月份導航
2. **主體**: 日曆網格
   - 每個日期格子顯示：可銷售數量、價格、狀態顏色
3. **側邊面板**: 批量編輯功能
   - 日期範圍選擇
   - 可銷售數量設置
   - 價格設置（平日/假日）
   - 開放/關閉狀態
4. **底部**: 保存按鈕、快速操作按鈕
