## 概述

本指南詳細說明了歐堡商務汽車旅館訂房系統中訂單的完整處理流程。從客戶提交訂房申請到入住完成，每個環節都有明確的操作步驟和責任分配。

### 核心原則

訂單處理應遵循以下核心原則：**及時回應、清晰溝通、準確記錄、確保完成**。員工應在每個環節及時更新訂單狀態，並通過郵件或電話與客戶保持溝通。

---

## 訂單生命週期

### 完整流程圖

```
客戶訂房 → 待確認 → 已確認 → 已匯款 → 已付款 → 已入住 → 已完成
                ↓
            (可選) 已取消
```

### 流程說明

訂單從客戶在線提交開始，系統自動創建「待確認」狀態的訂單。員工需要在後台管理系統中逐步推進訂單狀態，直到客戶入住並完成住宿。

## 訂單狀態說明

### 1. 待確認（Pending）

**觸發條件**：客戶在線提交訂房表單  
**持續時間**：1-24 小時  
**員工操作**：審核訂單信息，確認房間可用性

| 檢查項目 | 說明 |
|---------|------|
| 客戶信息完整性 | 確保姓名、電話、郵件都已填寫 |
| 日期有效性 | 確保入住日期早於退房日期 |
| 房間可用性 | 確認選定房型在指定日期可用 |
| 價格準確性 | 驗證報價與系統設置一致 |

**操作步驟**：
1. 在後台管理系統中打開「訂單管理」頁面
2. 查看「待確認」訂單列表
3. 點擊訂單卡片查看詳細信息
4. 驗證所有信息無誤後，點擊「確認訂房」按鈕
5. 系統自動發送確認郵件給客戶

### 2. 已確認（Confirmed）

**觸發條件**：員工點擊「確認訂房」按鈕  
**持續時間**：1-7 天  
**員工操作**：等待客戶選擇支付方式

| 支付方式 | 說明 | 後續步驟 |
|---------|------|---------|
| 銀行轉帳 | 客戶通過銀行轉帳支付 | 等待轉帳確認 |
| 現場支付 | 客戶在入住時現場支付 | 等待入住 |

**操作步驟**：
1. 訂單進入「已確認」狀態後，員工應通過郵件或電話聯繫客戶
2. 告知客戶兩種支付方式選項
3. 等待客戶選擇支付方式
4. 在後台系統中記錄客戶選擇的支付方式

### 3. 已匯款（Bank Transfer Pending）

**觸發條件**：客戶選擇銀行轉帳支付方式  
**持續時間**：1-3 天  
**員工操作**：監控銀行帳戶，確認轉帳收款

**銀行帳戶信息**：
- 帳戶名稱：歐堡商務汽車旅館有限公司
- 銀行代碼：[待填寫]
- 帳號：[待填寫]

**操作步驟**：
1. 客戶選擇銀行轉帳後，系統自動發送轉帳指示郵件
2. 員工定期檢查銀行帳戶進帳
3. 確認收到轉帳後，在後台系統中找到相應訂單
4. 點擊「確認銀行轉帳」按鈕，輸入轉帳後五碼
5. 系統自動更新訂單狀態為「已付款」

### 4. 已付款（Paid）

**觸發條件**：員工確認收到支付（銀行轉帳或現場支付）  
**持續時間**：1-7 天  
**員工操作**：準備房間，等待客戶入住

**準備工作**：
- 確保房間清潔衛生
- 檢查房間設施是否正常
- 準備客戶特殊需求（如加床、嬰兒床等）
- 準備歡迎信息或特殊禮物

**操作步驟**：
1. 訂單進入「已付款」狀態後，員工應確認房間已準備就緒
2. 通過郵件或電話提醒客戶入住時間和相關信息
3. 準備入住手續所需的文件和鑰匙
4. 在系統中記錄房間準備狀態

### 5. 已入住（Checked In）

**觸發條件**：客戶辦理入住手續  
**持續時間**：根據訂單天數  
**員工操作**：提供住宿服務，處理客戶需求

**入住手續**：
- 驗證客戶身份
- 交付房鑰匙
- 說明房間設施使用方法
- 提供酒店服務信息
- 收集客戶聯繫方式（便於緊急聯繫）

**服務期間**：
- 及時回應客戶需求
- 提供 24 小時客服支持
- 記錄客戶投訴或特殊要求
- 確保房間清潔和設施正常

### 6. 已完成（Completed）

**觸發條件**：客戶辦理退房手續  
**持續時間**：結束  
**員工操作**：結算費用，收集反饋

**退房手續**：
- 檢查房間狀況
- 結算額外費用（如超時、損壞等）
- 收回房鑰匙
- 感謝客戶並邀請再次光臨
- 發送退房確認郵件

**後續工作**：
- 清潔和檢查房間
- 記錄房間狀況和損壞情況
- 收集客戶評價和反饋
- 歸檔訂單文件

### 7. 已取消（Cancelled）

**觸發條件**：客戶主動取消或員工因特殊原因取消  
**退款政策**：根據取消時間決定

| 取消時間 | 退款比例 | 說明 |
|---------|---------|------|
| 入住前 7 天以上 | 100% | 全額退款 |
| 入住前 3-7 天 | 50% | 退款 50% |
| 入住前 1-3 天 | 0% | 不退款 |

**操作步驟**：
1. 客戶提出取消申請或員工發起取消
2. 在後台系統中找到訂單
3. 點擊「取消訂單」按鈕
4. 系統自動計算退款金額
5. 發送取消確認和退款說明郵件給客戶
6. 記錄取消原因

### 訂單管理後台介面

| 已確認 | 已確認但未支付的訂單 | 🟡 中 |
| 待付款 | 等待銀行轉帳確認的訂單 | 🔴 高 |
| 今日入住 | 今天入住的訂單 | 🔴 高 |

#### 訂單卡片信息

每個訂單卡片顯示以下信息：

- **狀態徽章**：訂單當前狀態（待確認、已確認等）

- **訂單編號**：系統自動生成的唯一編號
- **訂單金額**：總價格（NT$）
- **入住倒計時**：距離入住的天數
- **房型**：選定的房型名稱
- **日期**：入住和退房日期
- **人數**：入住人數
- **聯絡方式**：電話和郵件

#### 快速操作按鈕

根據訂單狀態，卡片上會顯示不同的操作按鈕：

| 狀態 | 可用按鈕 | 功能 |
|------|---------|------|
| 待確認 | 確認訂房 | 審核無誤後確認訂單 |
| 已確認 | 銀行轉帳、現場支付 | 記錄客戶選擇的支付方式 |
| 待付款 | 確認銀行轉帳 | 確認收到轉帳並輸入後五碼 |

### 日常操作流程

#### 上班第一件事：檢查待確認訂單

1. 登錄後台管理系統
2. 點擊「訂單管理」選項卡
3. 點擊「待確認」快速篩選按鈕
4. 逐個檢查待確認訂單
5. 對於每個訂單，驗證信息無誤後點擊「確認訂房」

#### 上午工作：跟進支付狀態

1. 點擊「待付款」快速篩選按鈕
2. 查看是否有銀行轉帳進帳
3. 對於已收到轉帳的訂單，點擊「確認銀行轉帳」按鈕
4. 輸入轉帳後五碼確認
5. 系統自動發送確認郵件給客戶

#### 下午工作：準備今日入住

1. 點擊「今日入住」快速篩選按鈕
2. 確認房間已清潔和準備就緒
3. 準備入住手續所需的文件
4. 通過電話或郵件提醒客戶入住時間

#### 晚間工作：處理退房和反饋

1. 檢查是否有客戶退房
2. 辦理退房手續並結算費用
3. 更新訂單狀態為「已完成」
4. 記錄客戶反饋和特殊事項

## 支付處理流程

### 銀行轉帳流程

#### 第 1 步：客戶選擇銀行轉帳

客戶在訂單確認後，選擇銀行轉帳作為支付方式。系統自動發送以下信息給客戶：

- 銀行帳戶名稱
- 銀行代碼
- 帳號
- 轉帳金額
- 轉帳截止日期

#### 第 2 步：客戶進行銀行轉帳

客戶通過網路銀行或臨櫃轉帳將金額轉入酒店帳戶。

#### 第 3 步：員工確認轉帳

1. 定期檢查銀行帳戶進帳
2. 確認轉帳金額和客戶信息匹配
3. 在後台系統中找到相應訂單
4. 點擊「確認銀行轉帳」按鈕
5. 輸入轉帳後五碼（作為確認憑證）
6. 點擊「確認」按鈕
7. 系統自動更新訂單狀態為「已付款」
8. 系統自動發送確認郵件給客戶

### 現場支付流程

#### 第 1 步：客戶選擇現場支付

客戶在訂單確認後，選擇現場支付作為支付方式。系統自動發送以下信息給客戶：

- 入住時需要支付的金額
- 接受的支付方式（現金、信用卡等）
- 入住時間和地點

#### 第 2 步：客戶入住時支付

客戶在辦理入住手續時支付房費。

#### 第 3 步：員工記錄支付

1. 收取客戶支付的金額
2. 開具收據
3. 在後台系統中找到相應訂單
4. 點擊「標記入住」按鈕
5. 系統自動更新訂單狀態為「已入住」

## 異常情況處理

### 情況 1：客戶未按時支付

**症狀**：訂單在「已確認」狀態超過 7 天仍未選擇支付方式

**處理步驟**：
1. 通過電話或郵件聯繫客戶
2. 確認客戶是否仍需要預訂
3. 提醒客戶選擇支付方式
4. 如果客戶不回應，可考慮取消訂單

### 情況 2：銀行轉帳金額不符

**症狀**：收到的轉帳金額與訂單金額不符

**處理步驟**：
1. 不要確認轉帳
2. 通過電話或郵件聯繫客戶
3. 確認是否為客戶錯誤或特殊情況
4. 要求客戶補付差額或退款
5. 確認無誤後再更新訂單狀態

### 情況 3：客戶臨時取消

**症狀**：客戶在入住前要求取消訂單

**處理步驟**：
1. 確認客戶取消意願
2. 在後台系統中點擊「取消訂單」按鈕
3. 系統自動計算退款金額
4. 通過郵件發送取消確認和退款說明
5. 按照退款政策處理退款

### 情況 4：客戶信息不完整

**症狀**：訂單缺少必要信息（如電話或郵件）

**處理步驟**：
1. 通過現有聯繫方式聯繫客戶
2. 要求客戶提供缺失的信息
3. 在系統中更新客戶信息
4. 確認信息完整後再確認訂單

### 情況 5：房間突然不可用

**症狀**：訂單確認後，房間因故障或維修變得不可用

**處理步驟**：
1. 立即通過電話聯繫客戶
2. 解釋情況並道歉
3. 提供替代方案（其他房型、改期等）
4. 如果客戶同意替代方案，更新訂單
5. 如果客戶要求取消，按照取消流程處理

## 常見問題

### Q1：訂單確認後多久應該聯繫客戶？

**A**：建議在訂單確認後 24 小時內聯繫客戶，確認其支付方式選擇。如果客戶在 7 天內未選擇支付方式，應再次提醒。

### Q2：銀行轉帳確認需要多久？

**A**：通常銀行轉帳在 1-3 個工作日內到帳。建議在客戶轉帳後 3 個工作日內確認轉帳。

### Q3：如何處理客戶的特殊需求？

**A**：在訂單卡片上記錄客戶的特殊需求（如加床、高樓層等），並在房間準備時予以滿足。如果無法滿足，應提前通知客戶並提供替代方案。

### Q4：訂單可以修改嗎？

**A**：訂單確認後，日期和房型通常不能修改。如果客戶需要修改，建議取消原訂單並創建新訂單。

### Q5：如何處理客戶投訴？

**A**：認真傾聽客戶投訴，記錄詳細信息，並立即向主管報告。根據投訴內容提供解決方案，如補償、免費升級等。

### Q6：訂單刪除後可以恢復嗎？

**A**：刪除訂單是永久操作，無法恢復。請謹慎操作，確保訂單確實需要刪除。
